Introduccion

KDD se conoce como Knowledge Discovery in Database y se define como un método secuencial e iterativo para encontrar, transformar y refinar datos y patrones significativos a partir de una base de datos sin procesar para poder utilizarlos en diferentes dominios o aplicaciones.


Desarrollo
Pasos involucrados en el proceso KDD:

Paso 1: Limpieza de datos

La limpieza de datos se define como la eliminación de datos ruidosos e irrelevantes de la recopilación.

  1. Limpieza en caso de valores faltantes .
  2. Limpieza de datos ruidosos , donde el ruido es un error aleatorio o de varianza.
  3. Limpieza con detección de discrepancias de datos y herramientas de transformación de datos .
Paso 2: Integración de datos

La integración de datos se define como datos heterogéneos de múltiples fuentes combinados en una fuente común (DataWarehouse)

  1. Integración de datos utilizando herramientas de migración de datos .
  2. Integración de datos mediante proceso ETL (Extract-Load-Transformation).
Paso 3: Selección de datos

La selección de datos se define como el proceso en el que los datos relevantes para el análisis se deciden y recuperan de la recopilación de datos

  1. Selección de datos mediante red neuronal
  2. Selección de datos mediante árboles de decisión .
  3. Selección de datos mediante Naive bayes .
Paso 4: Transformación de datos

La transformación de datos se define como el proceso de transformación de datos en la forma adecuada requerida por el procedimiento de minería.

  1. Mapeo de datos : asignación de elementos desde la base de origen hasta el destino para capturar transformaciones.
  2. Generación de código : Creación del programa de transformación real.
Paso 5: Minería de datos

La minería de datos se define como técnicas inteligentes que se aplican para extraer patrones potencialmente útiles.

  1. Transforma los datos relevantes de la tarea en patrones .
  2. Decide el propósito del modelo usando clasificación o caracterización .
Paso 6: Evaluación de patrones

La evaluación de patrones se define como la identificación de patrones estrictamente crecientes que representan el conocimiento basado en medidas dadas.

  1. Encuentre la puntuación de interés de cada patrón.
  2. Utiliza resumen y visualización para que los datos sean comprensibles para el usuario.
Paso 7: Representación del conocimiento

La representación del conocimiento se define como una técnica que utiliza herramientas de visualización para representar los resultados de la minería de datos.

  1. Generar informes .
  2. Generar tablas .
  3. Generar reglas discriminantes, reglas de clasificación, reglas de caracterización , etc.

Ejemplo

Entonces, sigue adelante y recolecta piedras, conchas, monedas o cualquier artefacto que pueda estar en el lecho del río. Pero eso también trae suciedad y otros objetos no deseados, de los que deberá deshacerse para tener los objetos listos para su uso posterior.


Ventajas
  • Descripción de los datos en términos familiares para el usuario.
  • Documentación automática de los datos y pasos de procesamiento.
  • La reutilización de las aplicaciones en diferentes bases de datos KDD.
  • Intercambiabilidad de conocimiento acerca de aplicaciones exitosas de KDD

Desventajas
  • Costo: el almacenamiento de volúmenes de datos masivos y en constante desarrollo conlleva un costo inicial obvio

Conclusion

La minería de datos como parte del proceso de KDD constituye una herramienta estratégica para las empresas y de apoyo en la toma de decisiones. El método KDD nos ayudará a encontrar conocimiento útil, válido, relevante y nuevo sobre un fenómeno o actividad mediante algoritmos eficientes, dadas las crecientes órdenes de magnitud en los datos
